#### Thread(Runnable target)

构造方法。接收一个实现了 `Runnable` 接口的任务。

#### run()

实例方法。线程启动后要执行的代码（需要重写）。

直接调用 `run()` 方法并不会开启新的线程，而是普通的方法调用，会阻塞后面代码的执行。

#### start()

实例方法。启动一个新线程，并调用 `run()` 方法。

Java 的线程本质上就是操作系统的线程。因为 Java 本身无法直接创建线程，它需要通过调用本地方法 `start0()`，由底层 JVM 使用 C++ 的函数来完成线程的创建。具体来说，JVM 会根据不同的操作系统，调用对应的原生线程 API 创建并启动一个本地线程（为了实现 JVM 的跨平台特性），最终由该线程执行 `run()` 方法中的代码。

#### yield()

静态方法。告诉线程调度器自己愿意主动让出 CPU 使用权，让其它线程先执行。

但是否真正让出，由调度器根据系统当前的状况决定——如果资源充足，可能并不会真正让出。

#### sleep()

静态方法。让当前线程暂停执行指定时间（以毫秒为单位），期间会释放 CPU，但不会释放锁，会抛出检查型异常。

```java
try {
	Thread.sleep(1000);
} catch (InterruptedException e) {
	e.printStackTrace();
}
```

更推荐的写法是使用 `TimeUnit` 枚举，这种写法更清晰，无需换算单位：

```java
try {
	TimeUnit.SECONDS.sleep(1);
} catch (InterruptedException e) {
	e.printStackTrace();
}
```

#### join()

实例方法。让当前线程等待目标线程执行完毕后，再继续往下执行。

#### setDaemon(boolean on)

实例方法。设置此线程为守护线程或用户线程。

用户线程（User Thread）：负责执行业务逻辑的核心线程，程序的主要功能依赖它们来完成。新创建的线程默认都是用户线程。只要有任何一个用户线程未执行完毕，JVM 就不会退出；只有当所有用户线程都执行结束后，JVM 才会终止运行。

守护线程（Daemon Thread）：当所有用户线程结束后，JVM 才会自动退出，并强制终止所有守护线程，无论它们的任务是否完成。因此，不应在守护线程中执行重要的任务。守护线程通常用于执行后台服务工作，为其他线程提供支持，例如垃圾回收（GC）。需要注意的是，守护线程必须在调用 `start()` 方法之前设置。

#### currentThread()

静态方法。返回当前执行线程的对象。
