#### 线程是什么？

你可以把进程想象成一家公司，线程就是公司里的员工。员工们可以一起使用公司的资源，比如办公室、电脑、文件等（对应程序的内存、文件等资源），但每个人可以负责不同的工作，同时并行完成任务。公司本身不会干活，真正执行工作的，是里面的员工，也就是线程。

你也可以将进程比作一个工厂，而线程就是工厂中的工人。多个工人可以协同工作，共享工厂里的工具和原料（即资源），以完成各项任务。一个工厂至少需要有一个工人来运转，如果没有工人，厂房即使具备所有资源，也无法进行生产。

你还可以把进程比作一栋房子，线程则是住在房子里的人。真正执行任务的是这些人，而不是房子本身。一栋房子（进程）可以住一个人（单线程），也可以住很多人（多线程），每个人都可以同时做不同的事情，实现并发完成任务。房子拥有自己的资源，比如水、电、家具等，类似于进程中的内存、文件等资源，这些资源由住在其中的人（线程）共同使用，各自协作完成工作。

线程是程序执行和 CPU 调度的最小单位。在现代操作系统中，所谓的“进程调度”，本质上就是“线程调度”，即操作系统以线程作为调度的基本单位。你编写的代码最终都是由线程来执行的——没有线程，代码就无法运行。

当创建一个进程时，系统会默认启动一个主线程，随后该进程可以根据需要创建多个线程。操作系统中的调度器（Scheduler）会选择具体的线程来分配 CPU 时间片，而不是直接调度整个进程。

线程不能独立存在，必须依附于进程之中。进程只是作为资源容器存在，负责为线程提供运行所需的环境；而线程则是实际在这些资源上执行任务的基本单位，一个线程就是进程内部的一个独立执行流，负责实际执行任务。一个进程可以包含多个线程，这些线程之间共享进程的资源，例如内存空间、打开的文件和网络连接等。但每个线程都各自拥有执行栈、程序计数器和寄存器，彼此之间互不干扰。

#### 线程 vs 进程

定义：进程是操作系统进行资源分配和管理的基本单位，线程则是操作系统进行 CPU 调度和执行的基本单位。

隔离性：进程之间是相互独立的，一个进程的崩溃不会影响到其他进程。Chrome 浏览器的每个标签页就是一个进程，一个标签页崩溃了不影响别的标签页。而线程属于进程的一部分，运行时并不完全隔离。在 Java 中，如果某个线程抛出未捕获的异常，通常只会终止该线程，其它线程仍可照常运行，不会受到影响。而在 C++ 中，异常不会自动隔离到单个线程；如果一个线程抛出异常且未被捕获，将直接导致整个进程崩溃。

开销：创建进程的开销较大，因为操作系统需要为其分配独立的资源，如内存空间和文件句柄，并创建进程控制块（PCB）进行管理。进程终止时，这些资源还需被回收。进程切换时还需要保存和恢复完整的运行状态，处理过程较为复杂。相比之下，线程被称为“轻量级进程”，它们共享所属进程的资源，无需重复分配，从而显著降低了创建和销毁的成本。线程切换只需保存较少的上下文信息，因而速度更快，系统开销也更小。相比创建多个进程，线程的创建和上下文切换开销较低，适合处理高并发任务。

通信方式：进程之间的通信相对复杂，通常需要依赖操作系统提供的机制，如管道、消息队列、共享内存或套接字等。而线程属于同一进程，它们共享内存空间，因此线程之间的通信更为简单，可以直接通过共享变量或对象进行数据交换，无需额外的通信机制。

#### 多线程

多线程是指在单个进程中同时运行多个线程来实现任务的并发执行，每个线程可以执行相同或不同的任务。各个线程可以交替使用 CPU 资源，而在多核处理器上，不同线程甚至可以同时运行在多个核心上，从而实现真正的并行处理。

多线程技术使程序能够同时处理多个任务，避免因主线程阻塞而导致其它逻辑得不到机会执行，从而提升程序的运行效率。

多线程技术的主要目的是充分利用多核处理器的优势。对于单线程程序来说，无论 CPU 拥有多少个内核，它们始终只能使用其中的一个。而采用多线程后，程序可以将任务分配给多个内核并行处理，从而显著提高运行效率。

#### 并发和多线程的关系

首先，我们需要理解“并发”的概念。所谓并发（concurrent），是指在同一时间段内，系统可以同时（逻辑上）处理多个任务。多个任务看似同时进行，实际上是 CPU 以几十到几百毫秒的速度在各个任务之间高速切换，从而让用户产生这些任务在并行运行的错觉。换句话说，并发本质上是任务的交替执行，而不是在同一时刻真正意义上的同时执行。而并行（parallel）才指的是多个任务在多个处理器核心上同时运行。

并发可以通过多种方式实现，而多线程只是实现并发的一种方式，通过创建多个线程来达到同时执行多个任务的目的。

实际上，线程的数量通常远多于 CPU 的核心数。即使在多核 CPU 上，每个核心也只是在线程之间切换执行任务，因此这被称为多线程并发编程，而不是多线程并行编程。
