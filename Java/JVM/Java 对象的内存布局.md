### 对象头（Header）

对象头最小 12（开启指针压缩）或 16 字节，如果没有实例数据，那么一个对象就是 16 字节。

#### Mark Word（标记字段）

固定 8 字节。

不同类型的锁会导致对象头中的 Mark Word 内容不同。

在 Java 中，`hashCode()` 只有在你显式调用时，JVM 才会真正生成并将其写入对象头的 Mark Word 中。

如果在对象已经加锁的情况下再调用 `hashCode()`，就会由于 Mark Word 空间冲突而导致锁膨胀：偏向锁被撤销，锁状态升级为轻量级锁或重量级锁。这是因为偏向锁需要在 Mark Word 中写入线程 ID，而 hashCode 也占用相同的位置，两者无法共存。

一旦对象计算过 `hashCode()`，就不能再使用偏向锁，否则会造成数据覆盖问题。
当锁升级为轻量级或重量级锁后，Mark Word 中将不再保存线程 ID，而是用于存储指向线程栈帧中的锁记录（Lock Record）或指向重量级锁（Monitor）的指针。

#### Klass Pointer（类型指针）

开启压缩指针时是 4 字节，没开就是 8 字节。

指向方法区中类元数据（Class 模板）的指针，用于标识该对象的实际类型。JVM 通过这个指针可以定位到对象所属的类、继承结构、方法表等信息。

#### 数组长度（仅数组对象拥有）

数组对象在对象头中会额外多出一个字段，用于记录数组的长度。非数组对象则没有这个字段。

### 实例数据（Instance Data）

实例数据部分包含对象的所有成员变量。

父类的字段会排在子类字段之前。

### 对齐填充（Padding）

对象的大小必须是 8 个字节的倍数，不足会自动补齐。
