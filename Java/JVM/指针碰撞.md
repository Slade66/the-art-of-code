指针碰撞是 JVM 中一种依赖连续内存空间的高速对象分配技术，它常用于新生代的 Eden 区。

在指针碰撞分配中，JVM 维护一个指向连续内存区域下一个可用位置的指针。你可以把它看作是用于划分已用与空闲内存的标记。当需要分配新对象时，JVM 会检查从当前指针到内存区域末尾是否有足够空间；若条件满足，就将对象写入指针所指位置，并将指针向前移动对象大小的距离。

指针碰撞要求内存区域连续且规整，不能有碎片，因此无法与采用“空闲列表”（Free List）机制配合使用。所谓空闲列表，可以理解为 JVM 维护的一张“内存空位清单”，记录了所有可用内存块的位置和大小。当分配对象时，JVM 会遍历这张清单，找到一个合适的空位来放置新对象。这种方式适合内存碎片较多的情况，但分配效率较低。

相比之下，为了支持指针碰撞，JVM 通常结合具有“压缩”（Compacting）功能的垃圾回收器，如 Serial GC、Parallel Scavenge 和 G1 GC 的部分阶段。在压缩过程中，GC 会将所有“存活对象”移动到内存的一端，而“非存活对象”则直接被清除，其所占空间被回收。这种整理过程不仅清除了垃圾对象，还在内存中腾出一整块连续的可用区域，从而为指针碰撞提供理想的内存布局，大幅提升对象分配的效率。
