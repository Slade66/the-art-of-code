为了执行 Java 程序，Java 虚拟机（JVM）会将连续的内存空间划分为若干个具有不同功能的运行时数据区域。这些区域根据其生命周期和共享特性可以分为两类：

- 全局共享区域：堆和方法区在 JVM 启动时创建，在 JVM 关闭时销毁，其生命周期伴随着 JVM。这两个区域是所有线程共享的，用于存储所有线程都能访问的数据。

- 线程隔离区域：程序计数器、Java 虚拟机栈和本地方法栈是每个线程独有的。当一个新线程被创建时，JVM 会为该线程分配这些独立的内存区域；线程执行完毕后，这些区域会被自动回收，不同线程之间的数据互不影响。

根据 Java 虚拟机规范，JVM 的内存布局主要由以下几个部分构成：

## 程序计数器（The Program Counter Register）

程序计数器是 Java 虚拟机（JVM）运行时数据区域中一块很小的内存空间，唯一不会抛出 `OutOfMemoryError` 的区域。通过 `javac` 编译生成的 `.class` 文件中的字节码，每一条指令都有一个对应的地址。程序计数器总是指向当前线程正在执行的，或者即将要执行的字节码指令的地址，以便在线程切换后能够恢复回正确的执行状态。

Java 支持多线程并发执行。由于 CPU 会快速地在不同线程间切换，每个线程必须记录其当前执行到的指令位置，用于在线程重新获得 CPU 时间片时，能够准确地恢复到之前的执行点继续运行。因此，程序计数器是线程私有的。

如果当前线程正在执行的是一个本地方法，那么该线程的程序计数器的值为空（undefined）。

## Java 虚拟机栈（Java Virtual Machine Stacks）

Java 虚拟机栈（也称为方法栈）的主要作用是支持方法的执行。用于存储栈帧（方法执行时所需的数据），管理方法调用的顺序，追踪整个方法调用的链条。每当一个 Java 方法被调用时，一个新的栈帧就会被创建并压入到当前线程的虚拟机栈中。当方法执行完毕并返回时，对应的栈帧就会被弹出。

每个栈帧都对应一次具体的方法调用。每个方法从调用开始到执行完成的过程，都对应着一个栈帧在虚拟机栈里从入栈到出栈的过程。不同调用（即使是同一个方法的递归调用）拥有独立的栈帧，保证了局部变量等的隔离性。当前正在执行的方法对应的栈帧位于该线程 JVM 栈的顶部，称为“当前栈帧”。

栈帧的内容：
- 局部变量表（Local Variable Table）：
	- 用于存储方法参数和方法内部定义的局部变量的内存空间，它的容量以槽（Slot）为单位。
	- 一个 Slot 能存放一个 `boolean`、`byte`、`char`、`short`、`int`、`float`、`reference`（对象引用）或 `returnAddress` 类型的数据。对于 `long` 和 `double` 这种 64 位的数据类型，则需要两个连续的 Slot 来存储。
	- 对于实例方法（非 `static` 方法），其局部变量表的第 0 个槽位（Slot）固定用于存放指向当前对象实例的 `this` 引用。方法参数和内部定义的局部变量从索引 1 开始依次存储。
	- 局部变量表所需的内存空间在编译期间就已经确定了，在方法运行期间不会改变。
	- Slot 是可以复用的。当一个局部变量离开了其作用域（例如，它定义在一个代码块 `{...}` 中，并且执行流离开了该代码块），它所占用的一个或多个槽位就可以被方法中稍后声明的新局部变量所复用，这有助于节省栈帧空间。
- 操作数栈（Operand Stack）：
	- 用于在执行方法时临时存储运算所需的操作数及产生的中间结果，是字节码指令之间进行数据交换的工作台。
	- 操作数栈可以存储 JVM 定义的各种数据类型的值。对于 `boolean`, `byte`, `short`, `char` 类型的值，在压入操作数栈之前，通常会被转换为 `int` 类型进行处理。`long` 和 `double` 类型的值占用两个栈单位（Slot），其它类型占用一个。
	- 操作数栈的最大深度在编译期间就已经计算出来了，记录在方法字节码的属性中，运行时其深度不会动态变化。
- 动态链接（Dynamic Linking）：
	- 这个区域存储着当前方法执行时所需的外部符号引用，用于指向被调用的方法和引用的字段。
	- 这些符号引用具体包括：被引用类的全限定名（例如 `com/example/MyClass`）、被引用方法或字段的名称（例如 `methodA`），以及它们的描述符（例如方法描述符 `()V` 表示无参且返回 `void`）。
	- 你可以将这个区域视为一个符号表，其中的每一条记录都对应着当前方法需要使用的一个外部资源。
	- 当程序执行到需要调用一个方法或访问一个字段的指令时，JVM 会根据当前栈帧的动态链接区域中存储的符号引用，在运行时常量池中查找对应的项。如果找到了匹配的符号引用，JVM 就会解析这个符号引用，获取到被引用方法或字段的直接引用。
- 方法返回地址（Return Address）：
	- 当一个方法被执行完毕后，程序需要知道回到哪里继续执行，方法返回地址就是用来记录这个返回位置的。
	- 当一个新方法被调用时，一个新的栈帧会被创建并压入虚拟机栈。在创建这个新栈帧的同时，调用者的调用指令的下一条指令的地址会作为被调用方法的返回地址保存起来。

JVM 栈的大小直接关系到线程能支持的方法调用最大深度。可以通过启动参数（如 `-Xss1m` 设为 1MB）调整这个大小，但这需要在调用深度和内存消耗之间权衡：栈过小易引发 `StackOverflowError`，而栈过大则会增加每个线程的内存占用。

## 本地方法栈（Native Method Stacks）

本地方法栈用于支持 JVM 调用本地方法，它为每个线程维护调用本地方法所需的栈帧。

本地方法是指在 Java 代码中使用 `native` 关键字声明的方法。这些方法没有具体的实现，它们的实现是用 C、C++ 等其它编程语言编写，并编译成本地库供 JVM 调用。

在 HotSpot 虚拟机中，为了简化实现并提高运行效率，Java 虚拟机栈（用于执行 Java 方法）与本地方法栈（用于执行 `native` 方法）被合并为一个统一的线程栈。这意味着每个线程实际上只有一个物理栈结构，它同时承担 Java 方法调用和本地方法调用的栈帧管理职责。当线程执行 Java 方法时，HotSpot 会在这个统一栈中压入遵循 Java 虚拟机规范的 Java 栈帧；而当线程调用 Native 方法时，则在同一个栈中压入符合本地平台调用约定的本地方法栈帧。虽然两类栈帧的结构不同，但它们共存于同一个线程栈中。
