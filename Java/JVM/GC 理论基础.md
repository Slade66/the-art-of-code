#### GC 有什么用？

垃圾回收（Garbage Collection, GC）是 Java 虚拟机（JVM）提供的一种自动内存管理机制，旨在简化内存管理的复杂性，提高程序的健壮性。与 C 或 C++ 等需要开发者手动分配和释放内存的语言不同，Java 开发者无需关心内存的生命周期管理，可以将更多精力集中在业务逻辑上，从而有效降低内存泄漏和野指针等问题的发生风险。JVM 的 GC 系统会自动追踪堆上分配的对象，并在对象不再被使用时回收其占用的内存，避免了手动管理内存带来的复杂性和风险。

**什么是垃圾？**

在程序运行过程中，凡是不再被任何存活引用指向的对象及其占用的内存，都被视为垃圾。

**什么是野指针？**

当指针变量指向了已被释放的内存地址，但该指针未被及时置为 `NULL`，仍尝试通过它访问该内存。这种行为可能导致程序崩溃，或误修改其它代码的数据，造成难以调试的错误。

**什么是内存泄漏？**

程序中不再使用的内存未被系统回收，且原本指向该内存的指针变量要么被重新赋值指向其它位置，要么超出作用域而消失，导致这块内存无法被访问或释放，从而与程序“失联”。内存泄漏在短时间运行的程序中影响较小，但对于需要长时间运行的程序，内存泄漏会导致内存占用持续增加，进而引发系统变慢，最终因资源耗尽而无法分配新内存，导致程序崩溃。
