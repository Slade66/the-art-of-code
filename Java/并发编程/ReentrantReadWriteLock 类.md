### 有什么用？

`ReentrantReadWriteLock` 是 Java 并发包（`java.util.concurrent.locks`）中提供的一种可重入的读写锁。它既能保证多个线程并发读取的高效性，又能在发生写操作时提供线程安全的保护。在“读多写少”的场景下，使用它能够显著提升系统的并发性能。默认是非公平锁，也可以通过构造器传参使用公平锁。

它解决了互斥锁（如 `synchronized` 或 `ReentrantLock`）在并发读取场景下的性能瓶颈：当多个线程同时读取共享资源时，无需互斥，可以并发执行，从而显著提升了系统的访问效率。

### 共享锁和独占锁

`ReentrantReadWriteLock` 内部实际上包含了两把子锁：
- ReadLock（读锁）：共享锁，允许多个线程同时获取（只要没有线程持有写锁）。
- WriteLock（写锁）：独占锁，同一时刻只能被一个线程持有。

通过分别对“读操作”和“写操作”加锁，读与写在内部能相互协调：
- 读锁可以同时被多个线程持有（读读共享）。
- 写锁只能同时由一个线程获取（写操作独占），且读锁与写锁不会同时共存。
- 当有线程写时，其它线程必须等待写操作完成后才能进行读取。

### 锁升级和锁降级

锁升级指的是一个线程已经持有了读锁，然后它又试图获取写锁，也就是想把“读”变成“写”。这样会导致死锁。因为写锁要求所有的读锁都被释放，而当前线程本身就持有着读锁且尚未释放，导致写锁始终无法获取，从而陷入死锁。

锁降级指的是在还持有写锁时，先获取读锁再释放写锁。在很多场景中，我们写完数据后还需要立刻读取这份刚写好的数据，并且希望其它读线程也能并发访问，同时又不想让其它写线程立即修改数据。当写操作完成后，降级为读锁能让其他读线程并发获取锁，而如果一直占着写锁，所有读线程都会被阻塞，影响系统吞吐量。锁降级很好地兼顾了“刚写完就要继续读”的需求与数据安全和可见性，同时最大化利用多线程的并发读能力，避免了“写锁占用过久”或“先释放写锁导致空窗期”这两种极端情况。
