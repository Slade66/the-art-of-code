### 有什么用？

传统的 `ArrayList` 在多线程环境下直接使用并不安全，而 `Collections.synchronizedList` 虽然能提供基本的线程安全保障，但在遍历时仍需显式加锁。否则，如果一个线程在遍历的同时，另一个线程对该列表进行修改，会抛出 `ConcurrentModificationException` 异常。

另一种线程安全的 List 是 `Vector`，这是一种已经过时的集合类。它通过在所有增、删、改、查等方法上使用 `synchronized` 关键字来实现线程安全。然而，这种做法在每次方法调用都必须先获取锁，导致在高并发场景下性能非常低下。

`CopyOnWriteArrayList` 是 Java 并发工具包里一个线程安全的 List，它可以作为 `ArrayList` 的并发版替代，不需要自己加锁。实际的业务场景中读操作远多于写操作，而读取本身并不会改变数据内容，因此对每次读取都加锁无疑会带来不必要的性能开销。`CopyOnWriteArrayList` 对读取操作采用了完全无锁的设计。更重要的是，写入操作不会妨碍正在进行的读取，只有在多个写入操作并发执行时才会进行互斥。这种机制使得读取操作的性能得到了显著的提升，特别适合那种读取频繁、写操作很少的场景。

### 怎么用？

**常用的方法**

- `add(E e)`：将指定的元素追加到列表的末尾。
- `add(int index, E element)`：在列表的指定位置插入指定的元素。
- `get(int index)`：返回列表中指定位置的元素。
- `set(int index, E element)`：用指定的元素替换列表中指定位置的元素。
- `remove(Object o)`：从此列表中移除第一次出现的指定元素（如果存在）。
- `remove(int index)`：移除列表中指定位置的元素。
- `size()`：返回此列表中的元素数。
- `isEmpty()`：如果此列表不包含任何元素，则返回 true。

### 底层原理

`CopyOnWriteArrayList` 采用了写时复制（Copy-On-Write）策略：当进行添加、修改或删除等写操作时，它不会直接修改原始数组，而是先创建一个新数组的副本，在副本上进行操作，完成后再通过原子方式将内部引用指向新的数组。

`CopyOnWriteArrayList` 非常适合读多写少的场景。它的读操作无需加锁，性能优越；但写操作采用写时复制策略，每次写入都需要复制整个数组，开销较大，不仅影响性能，还会占用额外内存。因此，在写操作频繁的场景下并不推荐使用。
