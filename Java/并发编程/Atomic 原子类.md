#### 什么是原子性？

“Atomic” 翻译成中文是“原子”。在化学中，原子是构成物质的最小单位，且在化学反应中不可再分。

原子性（Atomicity）是指一个操作是不可分割的。这意味着在执行该操作的过程中，不会被其他操作中断。要么操作完全成功执行，要么完全不执行，不存在中间状态。

#### 为什么需要 Atomic 原子类？

在多线程环境下，当多个线程同时访问和修改同一个共享变量时，由于线程的切换和执行顺序的不确定性，可能会导致出现竞态条件（Race Condition），从而使得结果与预期不符。

而 Atomic 原子类通过底层的硬件指令（通常是 Compare-and-Swap，简称 CAS）来保证操作的原子性，从而避免了使用传统的锁机制（如 synchronized）带来的性能开销和潜在的死锁风险。

#### 什么是竞态条件？

竞态条件是指多个线程在没有正确同步的情况下并发访问共享资源，导致执行结果不确定的现象；具体而言，程序的最终结果取决于这些并发操作执行时的相对顺序，而这个顺序通常是不可预测的。

**例一：多线程变量自增**

假设有两个线程同时对一个 `int` 类型的共享变量执行自增操作 (`count++`)。这个简单的操作在底层实际上包含了多个步骤：

1. 读取 `count` 的当前值。
2. 将值加 1。
3. 将新的值写回 `count`。

如果线程 A 在执行完第一步后被切换出去，而线程 B 完成了整个自增操作，然后线程 A 恢复执行并继续完成第二和第三步，那么最终 `count` 的值就只增加了 1，而不是预期的 2。

**例二：银行账户取款**

假设一个银行账户有 1000 元。现在有两个线程（比如代表两个 ATM 机或两个网上银行操作）同时尝试从此账户取款 500 元。

理想情况下，最终余额应该是 0 元。但如果发生竞态条件，可能会出现以下情况：

1. 线程 A 读取账户余额（1000 元）。
2. 线程 B 也读取账户余额（1000 元）。
3. 线程 A 计算新余额（1000 - 500 = 500 元），并将 500 元写入账户。
4. 线程 B 计算新余额（1000 - 500 = 500 元），也将 500 元写入账户。

最终结果是账户余额为 500 元，而不是预期的 0 元。银行损失了 500 元。这是因为线程 B 的操作覆盖了线程 A 的操作结果，两者都基于旧的余额（1000 元）进行计算。
