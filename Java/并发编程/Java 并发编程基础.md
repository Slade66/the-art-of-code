#### JUC 是什么？

JUC 是 java.util.concurrent 的缩写，是 Java 提供的一整套用于并发编程的工具包。

JUC 实际上是由多个包组成的：
- java.util.concurrent，提供了线程池、并发集合、阻塞队列、同步工具等
- java.util.concurrent.locks，提供了高级锁机制
- java.util.concurrent.atomic，提供了原子变量类

#### 如何停止一个线程

在 Java 中，安全地停止一个线程并不是直接“杀死”它。线程不应该被其它线程强行终止，因为这可能导致任务未完成或数据丢失。更合理的做法是通过某种机制通知线程该退出了，使它能在适当的时机完成必要的清理工作，并自行决定何时停止。常见的两种实现方式如下：

1. 中断法（推荐方式）：通过外部调用线程的 `interrupt()` 方法，向目标线程发送中断信号。线程内部需要定期调用检查 `isInterrupted()` 来检查中断请求，并自行决定何时退出。
2. 标志位控制法：通过设置一个 `volatile` 类型的布尔变量或使用 `AtomicBoolean` 类，由线程在执行过程中定期检查该标志，若为真则退出执行。

#### Object.wait() 和 Thread.sleep() 的区别

所属类不同：`sleep()` 是 `Thread` 类的方法；`wait()` 是 `Object` 类的方法。

是否释放锁：`sleep()` 不会释放当前线程持有的对象锁；wait() 会立即释放当前线程持有的对象锁。

是否必须在同步块中使用：`sleep()` 可以在任何地方调用，无需处于同步代码块中；`wait()` 必须在 `synchronized` 修饰的同步方法或代码块中调用，否则会抛出 `IllegalMonitorStateException`。

唤醒方式不同：`sleep()` 到达指定时间后自动恢复执行，无需外部干预。`wait()` 必须由其他线程调用 `notify()` 或 `notifyAll()` 显式唤醒，否则会一直处于等待状态。

作用机制不同：`sleep()` 是线程自己告诉操作系统“我先不干了”，主要用于线程的定时暂停；`wait()` 是线程告诉调度器“我等别人通知”，用于线程间的协作通信。
