`LockSupport` 是 JDK 提供的一个底层并发工具类，用来挂起（阻塞）和唤醒线程，是实现 AQS、各种锁（如 `ReentrantLock`、`CountDownLatch`）等并发组件的基础。

可以先唤醒后阻塞，这是因为 `LockSupport` 采用的是“凭证机制”。每个线程内部都隐含一个“许可证（permit）”，初始状态是没有的。当调用 `LockSupport.unpark(Thread t)` 时，会为指定线程发放一个许可证：如果该线程尚未阻塞，许可证会被记录下来；如果线程已经阻塞，则立即唤醒它。调用 `LockSupport.park()` 会尝试挂起当前线程，但在挂起前会先检查是否持有许可证：如果有，就直接消耗掉，不阻塞；如果没有，才会真正进入阻塞状态。需要注意的是：每个线程最多只能持有一个许可证，多次调用 `unpark()` 不会累计多个许可证。

底层使用的是 `Unsafe.park()` 和 `Unsafe.unpark()` 方法。

`park()` 的时候可以被打断，但不会抛出异常，需要自己检测。

可能出现伪唤醒，JVM 可能在没有 `unpark()` 或 `interrupt()` 的情况下唤醒线程。
