虚拟线程（Virtual Thread）是一种由 JVM 管理、运行在平台线程之上的轻量级线程，相当于其它编程语言中的“协程”。它旨在降低频繁线程切换带来的开销，并显著提升系统的并发能力。

传统的原生线程（也称平台线程）创建成本高，每个线程约占 1MB 内存，且线程切换需要在用户态与内核态之间转换，开销较大，系统通常只能支持几万个原生线程。相比之下，虚拟线程运行在平台线程之上，属于用户态线程，创建成本极低、内存占用仅为几 KB，且线程切换无需进入内核态，大幅降低了资源消耗，虚拟线程的数量可达百万级。

如果把线程比作插座，那虚拟线程就相当于一个插排，可以在同一个平台线程上承载多个虚拟线程，就像一个插排能同时连接多个电器一样，提高了资源利用率。

虚拟线程非常适合处理 IO 阻塞型任务。在任务阻塞期间，它可以主动释放 CPU，让其它任务先执行，从而提升整体系统的并发效率。但对于 CPU 密集型任务来说，虚拟线程并不适用，因为这类任务一直占用 CPU，如果频繁切换反而会带来额外开销，得不偿失。
