- **报错：**`test.go:3:8: read ...\lib.go: unexpected NUL in input`
- **原因：**
	- 在 Windows PowerShell 环境下创建文件时，用了类似 `echo "package ..." > lib.go` 的命令。
	- **PowerShell (5.x版本)** 的 `>` 重定向符号默认使用 **UTF-16 (LE)** 编码。
	- UTF-16 编码的特点是：每个英文字母后面都会跟一个 `0x00` (NUL 空字节)。比如 `p` 会被存成 `p\0`。
	- **Go 语言编译器**只认 **UTF-8** 编码。当它读取 `lib.go` 时，读完第一个字母紧接着读到一个空字节（NUL），它就懵了，直接报错。
- **解决：**
	- 用 VS Code 打开报错的那个 `.go` 文件，检查右下角编码，选择 **"UTF-8"**，然后覆盖保存。
